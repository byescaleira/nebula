# Decisions

Architecture decisions for the **Nebula 26 foundation** (from-scratch, starting 2026-07-18).
See `CLAUDE.md` for the binding guidelines and `VERSIONING.md` for the versioning policy.

| Date | Decision | Context | Consequences | Status |
|------|----------|---------|--------------|--------|
| 2026-07-18 | From-scratch Nebula foundation, single SPM target with internal folders | Sibling of Cosmos (SwiftUI design system). Foundation concerns (logging/errors/extensions/formatting/measure) need a Sendable-first, Apple-aligned, Swift-6-clean home. Sub-products would fragment the API and break the "one import" rule. | Single `Nebula` target; one `import Nebula`; internal folders (`Nebula.swift` top-level, `Logging`/`Errors`/`Extensions/{DateTime,String,Number,Primitive,Collection,Codable,DataURL}`/`Standardize`/`Measure`/`Nebula.docc`) are physical, not module, boundaries. No `NebulaLogging`/`NebulaExtensions` sub-products. No `Core/`, `Concurrency/`, or `Formatting/` dirs shipped. | Decided |
| 2026-07-18 | Multiplatform — all 5 platforms at `.v26`; `swift-tools-version: 6.3`; `swiftLanguageModes: [.v6]` | User wants every subsystem to behave well per platform on iOS/macOS/tvOS/watchOS/visionOS 26. Dual Xcode 26.4+ / Xcode 27 build requirement. | `Package.swift` targets all 5 at `.v26`; OS-27-only SDK symbols compile-gated `#if swift(>=6.4)` (graceful fallback on Swift 6.3, enabled on Swift 6.4). Xcode 26.0–26.3 (Swift 6.2) will NOT parse a 6.3 manifest — require Xcode 26.4+. | Decided |
| 2026-07-18 | UIKit-free package (Foundation-only) | Nebula is a foundation/architecture layer; UI is Cosmos's job. | No `import UIKit`/`UIColor`/`UIViewController`/`#if canImport(UIKit)`. Foundation + os + Synchronization + _Concurrency + CryptoKit (only behind `NebulaHashAlgorithm`). | Decided |
| 2026-07-18 | `os.Logger` over `swift-log` (zero third-party deps) | User: no third-party deps. `os.Logger` is native, Instruments-integrated, privacy-redacting (`OSLogMessage` preserves per-argument redaction via `@_semantics("oslog.requires_constant_arguments")`). | Thin Sendable façade over `os.Logger`/`OSSignposter`/`OSLogStore`. `swift-log` interop explicitly NOT provided. Document that `os.Logger` is the native alternative. | Decided |
| 2026-07-18 | Modern `FormatStyle` family over legacy `Formatter` subclasses | `NumberFormatter`/`DateFormatter`/`MeasurementFormatter`/`ByteCountFormatter`/`ISO8601DateFormatter` are non-Sendable, mutable, superseded by the Sendable `FormatStyle` family. Apple-aligned. | `NebulaStandards`/`NebulaDateFormat`/`NebulaDurationFormat`/`NebulaNumberFormatting` wrap only modern `FormatStyle` values. `ListFormatter` is the single exception (no FormatStyle replacement, not Sendable) — wrapped per-call, never cached. | Decided |
| 2026-07-18 | `os.Logger`/`os.OSSignposter` CANNOT be wrapped — expose the raw type (verified, compile-proven) | Empirically verified against the Xcode 27 Beta 3 toolchain: `os.Logger`'s level methods (`@_semantics("oslog.requires_constant_arguments")`) AND `log(level:_:)` (`oslog.log_with_level`) BOTH reject a forwarded `OSLogMessage` parameter ("argument must be a string interpolation"); `@_transparent`/`@inlinable`/`@_semantics` wrappers do NOT rescue it. `os.OSSignposter`'s signpost methods (`@_semantics("constant_evaluable")`) likewise reject forwarded `name: StaticString`/`message: SignpostMetadata` ("globalStringTablePointer builtin must be used only on string literals"). The literal MUST appear at the `os.Logger`/`os.OSSignposter` call site. | `NebulaLogger` exposes the underlying `os.Logger` as `public var osLogger` (the redaction-preserving path) and offers `String` convenience level methods that build the `OSLogMessage` literal AT the call site (legal). `NebulaSignposter` exposes `public var osSignposter` for `emitEvent`/`beginInterval`/`endInterval`/`withIntervalSignpost`; keeps Nebula-typed `makeSignpostID`/ID/state wrappers (no-literal ops). The vault note's "thin facade forwards `OSLogMessage`" design is corrected in its Corrections section. | Decided |
| 2026-07-18 | Sendable-everything + `Mutex`/`Atomic` (no `NSLock`/`DispatchQueue`/`nonisolated(unsafe)`) | Swift 6 mode v6 / zero concurrency warnings. User: "não vem com nslock … tudo conforme a nova concorrência, sem warnings." | All public value types `Sendable` by derived conformance; `@unchecked Sendable` only for thin wrappers around Apple-framework bridges (`NebulaMemoryLogHandler`); handlers `@Sendable`. Shared mutable state uses `Mutex<T>`/`Atomic<T>` from `import Synchronization` (SwiftStdlib 6.0 → iOS 18/macOS 15, below `.v26`). Region-based isolation (SE-0414) before `@unchecked`. | Decided |
| 2026-07-18 | Once-token pattern for idempotent setup | One-time registration/setup must be thread-safe without a lock primitive. | `static let` whose initializer side-effect runs exactly once via `swift_once` (Darwin `dispatch_once_f`). Side-effect-only form: `private static let _register: Void = { … }()`. The instance-scoped `NebulaOnce` wrapper is **deferred** (not shipped in 0.1.0) — tracked in `ROADMAP.md` → "Later". | Decided (mechanic) / Deferred (`NebulaOnce`) |
| 2026-07-18 | Versioning: Nebula N ↔ OS N (baseline Nebula 26) | User: totally versioned like SwiftUI, related to the OS version, starting with Nebula 26. | Library major == OS major; "since Nebula 26" == `@available(iOS 26, *)`; `@available` gates include all 5 platforms incl. `visionOS 26`; semver minor/patch within major; deprecation runway via `@available(*, deprecated, message:)`. See `VERSIONING.md`. | Decided |
| 2026-07-18 | Standardize + Measure unified under `NebulaStandards` + `NebulaMeasureConfiguration` + `NebulaSignposter` | Formatting and measurement are both Apple-native-FormatStyle/Clock facades; unifying avoids a fragmented surface. Signposts tie to the same `os.Logger` category as `NebulaLogger`. | `NebulaStandards` (formatting façade) + `NebulaMeasureConfiguration` (the 4th config struct — carries `measure(_:operation:)`/`bench(_:iterations:warmup:operation:)` ON the config, mirroring `NebulaLogConfiguration.log`; NO separate `NebulaMeasure` type) + `NebulaSignposter` (os.OSSignposter for Instruments, lives under Logging). `NebulaClock` is **deferred** — measure uses `any Clock<Duration>` directly (default `ContinuousClock`). | Decided (superseded naming: `NebulaMeasure` folded into `NebulaMeasureConfiguration`; `NebulaClock` deferred) |
| 2026-07-18 | Typed throws adoption: untyped `throws` default; `NebulaError` opt-in concrete `Failure` | SE-0413 evolution safety — declaring public APIs as `throws(NebulaError)` constrains future error types. | Public Nebula APIs use untyped `throws`; `NebulaError` is exposed as a concrete `Sendable: Error` for consumers that declare `throws(NebulaError)` / `Result<T, NebulaError>`. `throws(Never)` marks infallible. | Decided |
| 2026-07-18 | `NebulaError` envelope: `Error`/`LocalizedError`/`CustomNSError`/`Sendable`/`Hashable`; `underlying` boxed via `final class Box`; `NebulaErrorConfiguration` Sendable ONLY (NOT `Equatable`) | Verified compile risks: (1) a Swift `struct` cannot recursively contain itself — `underlying: NebulaError?` does NOT compile; (2) a struct with a stored `@Sendable` closure cannot conform to `Equatable` (mirrors `CosmosErrorConfiguration`). | `underlying: Box?` via `final class Box: Sendable, Hashable` (derived Sendable on a final class with a Sendable `let`, no `@unchecked`). `NebulaErrorConfiguration: Sendable` only; `NebulaErrorEvent: Sendable, Equatable` (all fields Equatable). `NebulaError.wrap(_:) -> Result<T, NebulaError>`. | Decided |
| 2026-07-18 | Vault-binding for research (mirror Cosmos) | Every research task must persist to `vault/`; root docs are source of truth; vault is synthesis. | Folders `01-fundamentos/` (10 foundation slugs), `03-padroes/` (`nebula-spm-architecture`, `nebula-swift6-concurrency`), `07-metodologia/`, `08-riscos/`, `02-decisoes/`. Kebab-case filenames, frontmatter, `[[wikilinks]]`. The `.swiftinterface` is authoritative ground truth (WebFetch has hallucinated availability tables). | Decided |
