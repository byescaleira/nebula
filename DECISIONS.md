# Decisions

Architecture decisions for the **Nebula 26 foundation** (from-scratch, starting 2026-07-18).
See `CLAUDE.md` for the binding guidelines and `VERSIONING.md` for the versioning policy.

| Date | Decision | Context | Consequences | Status |
|------|----------|---------|--------------|--------|
| 2026-07-18 | From-scratch Nebula foundation, single SPM target with internal folders | Sibling of Cosmos (SwiftUI design system). Foundation concerns (logging/errors/extensions/formatting/measure) need a Sendable-first, Apple-aligned, Swift-6-clean home. Sub-products would fragment the API and break the "one import" rule. | Single `Nebula` target; one `import Nebula`; internal folders (`Nebula.swift` top-level, `Logging`/`Errors`/`Extensions/{DateTime,String,Number,Primitive,Collection,Codable,DataURL}`/`Standardize`/`Measure`/`Nebula.docc`) are physical, not module, boundaries. No `NebulaLogging`/`NebulaExtensions` sub-products. No `Core/`, `Concurrency/`, or `Formatting/` dirs shipped. | Decided |
| 2026-07-18 | Multiplatform — all 5 platforms at `.v26`; `swift-tools-version: 6.3`; `swiftLanguageModes: [.v6]` | User wants every subsystem to behave well per platform on iOS/macOS/tvOS/watchOS/visionOS 26. Dual Xcode 26.4+ / Xcode 27 build requirement. | `Package.swift` targets all 5 at `.v26`; OS-27-only SDK symbols compile-gated `#if swift(>=6.4)` (graceful fallback on Swift 6.3, enabled on Swift 6.4). Xcode 26.0–26.3 (Swift 6.2) will NOT parse a 6.3 manifest — require Xcode 26.4+. | Decided |
| 2026-07-18 | UIKit-free package (Foundation-only) | Nebula is a foundation/architecture layer; UI is Cosmos's job. | No `import UIKit`/`UIColor`/`UIViewController`/`#if canImport(UIKit)`. Foundation + os + Synchronization + _Concurrency + CryptoKit (only behind `NebulaHashAlgorithm`). | Decided |
| 2026-07-18 | `os.Logger` over `swift-log` (zero third-party deps) | User: no third-party deps. `os.Logger` is native, Instruments-integrated, privacy-redacting (`OSLogMessage` preserves per-argument redaction via `@_semantics("oslog.requires_constant_arguments")`). | Thin Sendable façade over `os.Logger`/`OSSignposter`/`OSLogStore`. `swift-log` interop explicitly NOT provided. Document that `os.Logger` is the native alternative. | Decided |
| 2026-07-18 | Modern `FormatStyle` family over legacy `Formatter` subclasses | `NumberFormatter`/`DateFormatter`/`MeasurementFormatter`/`ByteCountFormatter`/`ISO8601DateFormatter` are non-Sendable, mutable, superseded by the Sendable `FormatStyle` family. Apple-aligned. | `NebulaStandards`/`NebulaDateFormat`/`NebulaDurationFormat`/`NebulaNumberFormatting` wrap only modern `FormatStyle` values. `ListFormatter` is the single exception (no FormatStyle replacement, not Sendable) — wrapped per-call, never cached. | Decided |
| 2026-07-18 | `os.Logger`/`os.OSSignposter` CANNOT be wrapped — expose the raw type (verified, compile-proven) | Empirically verified against the Xcode 27 Beta 3 toolchain: `os.Logger`'s level methods (`@_semantics("oslog.requires_constant_arguments")`) AND `log(level:_:)` (`oslog.log_with_level`) BOTH reject a forwarded `OSLogMessage` parameter ("argument must be a string interpolation"); `@_transparent`/`@inlinable`/`@_semantics` wrappers do NOT rescue it. `os.OSSignposter`'s signpost methods (`@_semantics("constant_evaluable")`) likewise reject forwarded `name: StaticString`/`message: SignpostMetadata` ("globalStringTablePointer builtin must be used only on string literals"). The literal MUST appear at the `os.Logger`/`os.OSSignposter` call site. | `NebulaLogger` exposes the underlying `os.Logger` as `public var osLogger` (the redaction-preserving path) and offers `String` convenience level methods that build the `OSLogMessage` literal AT the call site (legal). `NebulaSignposter` exposes `public var osSignposter` for `emitEvent`/`beginInterval`/`endInterval`/`withIntervalSignpost`; keeps Nebula-typed `makeSignpostID`/ID/state wrappers (no-literal ops). The vault note's "thin facade forwards `OSLogMessage`" design is corrected in its Corrections section. | Decided |
| 2026-07-18 | Sendable-everything + `Mutex`/`Atomic` (no `NSLock`/`DispatchQueue`/`nonisolated(unsafe)`) | Swift 6 mode v6 / zero concurrency warnings. User: "não vem com nslock … tudo conforme a nova concorrência, sem warnings." | All public value types `Sendable` by derived conformance; `@unchecked Sendable` only for thin wrappers around Apple-framework bridges (`NebulaMemoryLogHandler`); handlers `@Sendable`. Shared mutable state uses `Mutex<T>`/`Atomic<T>` from `import Synchronization` (SwiftStdlib 6.0 → iOS 18/macOS 15, below `.v26`). Region-based isolation (SE-0414) before `@unchecked`. | Decided |
| 2026-07-18 | Once-token pattern for idempotent setup | One-time registration/setup must be thread-safe without a lock primitive. | `static let` whose initializer side-effect runs exactly once via `swift_once` (Darwin `dispatch_once_f`). Side-effect-only form: `private static let _register: Void = { … }()`. The instance-scoped `NebulaOnce` wrapper is **deferred** (not shipped in 0.1.0) — tracked in `ROADMAP.md` → "Later". | Decided (mechanic) / Deferred (`NebulaOnce`) |
| 2026-07-18 | Versioning: Nebula N ↔ OS N (baseline Nebula 26) | User: totally versioned like SwiftUI, related to the OS version, starting with Nebula 26. | Library major == OS major; "since Nebula 26" == `@available(iOS 26, *)`; `@available` gates include all 5 platforms incl. `visionOS 26`; semver minor/patch within major; deprecation runway via `@available(*, deprecated, message:)`. See `VERSIONING.md`. | Decided |
| 2026-07-18 | Standardize + Measure unified under `NebulaStandards` + `NebulaMeasureConfiguration` + `NebulaSignposter` | Formatting and measurement are both Apple-native-FormatStyle/Clock facades; unifying avoids a fragmented surface. Signposts tie to the same `os.Logger` category as `NebulaLogger`. | `NebulaStandards` (formatting façade) + `NebulaMeasureConfiguration` (the 4th config struct — carries `measure(_:operation:)`/`bench(_:iterations:warmup:operation:)` ON the config, mirroring `NebulaLogConfiguration.log`; NO separate `NebulaMeasure` type) + `NebulaSignposter` (os.OSSignposter for Instruments, lives under Logging). `NebulaClock` is **deferred** — measure uses `any Clock<Duration>` directly (default `ContinuousClock`). | Decided (superseded naming: `NebulaMeasure` folded into `NebulaMeasureConfiguration`; `NebulaClock` deferred) |
| 2026-07-18 | Typed throws adoption: untyped `throws` default; `NebulaError` opt-in concrete `Failure` | SE-0413 evolution safety — declaring public APIs as `throws(NebulaError)` constrains future error types. | Public Nebula APIs use untyped `throws`; `NebulaError` is exposed as a concrete `Sendable: Error` for consumers that declare `throws(NebulaError)` / `Result<T, NebulaError>`. `throws(Never)` marks infallible. | Decided |
| 2026-07-18 | `NebulaError` envelope: `Error`/`LocalizedError`/`CustomNSError`/`Sendable`/`Hashable`; `underlying` boxed via `final class Box`; `NebulaErrorConfiguration` Sendable ONLY (NOT `Equatable`) | Verified compile risks: (1) a Swift `struct` cannot recursively contain itself — `underlying: NebulaError?` does NOT compile; (2) a struct with a stored `@Sendable` closure cannot conform to `Equatable` (mirrors `CosmosErrorConfiguration`). | `underlying: Box?` via `final class Box: Sendable, Hashable` (derived Sendable on a final class with a Sendable `let`, no `@unchecked`). `NebulaErrorConfiguration: Sendable` only; `NebulaErrorEvent: Sendable, Equatable` (all fields Equatable). `NebulaError.wrap(_:) -> Result<T, NebulaError>`. | Decided |
| 2026-07-18 | Vault-binding for research (mirror Cosmos) | Every research task must persist to `vault/`; root docs are source of truth; vault is synthesis. | Folders `01-fundamentos/` (10 foundation slugs), `03-padroes/` (`nebula-spm-architecture`, `nebula-swift6-concurrency`), `07-metodologia/`, `08-riscos/`, `02-decisoes/`. Kebab-case filenames, frontmatter, `[[wikilinks]]`. The `.swiftinterface` is authoritative ground truth (WebFetch has hallucinated availability tables). | Decided |
| 2026-07-18 | Lossy error mapping: `any Error` is not `Sendable` — consume at construction time | SE-0302: `any Error` is not `Sendable`; retaining it would break `NebulaError: Sendable`. | Mapping inits (`NSError`/`DecodingError`/`URLError`/`CocoaError`/`any Error`) drop the existential and keep only Sendable fragments (domain/code/strings/coding-path). `NebulaError.Box` gives ONE level of nesting; deeply chained `NSUnderlyingErrorKey` chains flatten to one level. Document loudly. | Decided |
| 2026-07-18 | `NebulaLogStoreExporter.Scope.system` / `.local()` macOS-only via `#if os(macOS)` / explicit per-platform `@available(<platform>, unavailable)` — NOT `@available(macOS 12, *)` alone | `@available(macOS 12, *)` on a declaration enables ALL platforms via the `*` fallback — it does NOT make a symbol macOS-only. `Store.h` API_UNAVAILABLE(ios, tvos, watchos); visionOS NOT listed (uncertain). | Use `#if os(macOS)` or explicit `@available(iOS, unavailable)`/`@available(tvOS, unavailable)`/`@available(watchOS, unavailable)`/`@available(visionOS, unavailable)`. visionOS conservatively gated unavailable (header does not prove availability). `NebulaLogStoreExporter.entries` copies into a Sendable `NebulaLogStoreEntry` (`OSLogEntry` is not Sendable). **Deferred** — `NebulaLogStoreExporter`/`NebulaLogStoreEntry` not shipped in 0.1.0; tracked in `ROADMAP.md` → "Later". | Decided (mechanic) / Deferred (type) |
| 2026-07-18 | `NebulaLogCategory`: String-rawValue struct with presets (NOT closed enum) | Flexible category extension without forking an enum; `os.Logger` takes a String category anyway. | `public struct NebulaLogCategory: Sendable, ExpressibleByStringLiteral` with presets `.networking`/`.persistence`/`.formatting`/`.measure`/`.concurrency`/`.general`. Consumers can add custom categories via string literal. | Proposed |
| 2026-07-18 | Default subsystem `"com.nebula.foundation"` placeholder; consumer overrides via `.withSubsystem` | Avoid cross-app category collisions in Console.app by encouraging consumer-supplied bundle-id-derived subsystem; but a static default is convenient. | Placeholder default; document loudly that consumers SHOULD override; `.withSubsystem(_:)` fluent builder. | Proposed |
| 2026-07-18 | `NebulaMemoryLogHandler` in main `Nebula` target (test/preview-only, documented) | A separate test-support module would violate the single-target rule; the handler's effective floor is iOS 18/macOS 15/visionOS 2 via `Mutex` (below `.v26`). | `public final class NebulaMemoryLogHandler: @unchecked Sendable` in `Logging/`; `Mutex<T>`-backed ring buffer; `snapshot()`; documented as test/preview-only. | Proposed |
| 2026-07-18 | `NebulaError` equality: auto-synthesized `Hashable` (includes `date`) | Two logically-identical errors with different timestamps compare non-equal under auto-synthesis; `NebulaErrorEvent` also carries `date`. | Auto-synthesized `==`/`hash(into:)` over all stored fields (including `date`); document that equality includes the timestamp. Revisit if consumers are surprised (deprecation path: custom `==` excluding `date`). | Proposed |
| 2026-07-18 | Process-wide `Mutex<Nebula*Config>` accessor + explicit-parameter DI (both) | Cosmos uses SwiftUI Environment injection; Nebula has no Environment — decide between global-Mutex default vs. explicit-parameter ergonomics. | Both paths supported: `NebulaErrorConfig.get()/set(_:)` (Mutex-backed) for ergonomics; explicit `NebulaErrorConfiguration` parameter for testability. Same pattern for log/standards/measure. | Proposed |
| 2026-07-18 | Ship `NebulaLocked`/`NebulaFlag`/`NebulaOnce` wrappers (Nebula-prefix consistency) | Wrappers cost `~Copyable` propagation; direct `Mutex`/`Atomic` use costs Nebula-prefix inconsistency. | Wrappers designed for `Concurrency/NebulaConcurrency.swift`; compile-validate `~Copyable`+`Sendable` synthesis against the exact Swift 6.4 toolchain before shipping (open risk). Hold as `let` globals or inside actors. **Deferred** — not shipped in 0.1.0 (no `Concurrency/` dir); Nebula uses `Mutex`/`Atomic` directly. Tracked in `ROADMAP.md` → "Later". | Deferred |
| 2026-07-18 | Ship NO `Localizable.xcstrings` at launch | Foundation emits developer-facing English log/error text; no user-facing strings by default. | `.process("Resources")` commented in `Package.swift`; uncomment when a catalog ships. CONTRIBUTING.md ties the catalog's presence to the resources block. | Proposed |
| 2026-07-18 | NO `swift-docc-plugin` dependency; use `xcodebuild docbuild` for CI doc generation | The binding constraint forbids third-party RUNTIME deps; the plugin is build-time, but keeping `dependencies: []` pristine is cleaner. | DocC catalog builds natively in Xcode 26/27; CLI generation via `xcodebuild docbuild` without a plugin dependency. | Proposed |
| 2026-07-18 | `NebulaError.Kind`: closed enum for v26 (deprecation path to open struct if needed) | A closed enum gives a stable classification taxonomy; an open struct lets consumers register custom kinds without forking. | `public enum Kind: String, Sendable, CaseIterable` (network/decoding/encoding/cocoa/file/validation/serialization/unknown). Reopen as `struct` if consumers need custom kinds (deprecation path). | Proposed |
| 2026-07-18 | `Decimal.rounded(toDecimalPlaces:)` via `NSDecimalRound` (C) + `NebulaDecimalRoundingRule` enum mapping 1:1 to `NSDecimalNumber.RoundingMode` | Decimal has NO value-level `rounded(_:)` (verified — only `Decimal.FormatStyle.rounded` returns a FormatStyle, not a Decimal). | Small `NebulaDecimalRoundingRule` enum (`.plain`/`.bankers`/`.down`/`.up`) mapping to `NSDecimalNumber.RoundingMode`; no `Double` round-trip, no `NSDecimalNumber` allocation. | Proposed |
| 2026-07-18 | `NebulaStandards` typed accessors only (NO polymorphic `.format(_:)` / `AnyFormatStyle`) | A polymorphic entry would lose type info and `AttributedString` variants; generic `<F: FormatStyle>` is sufficient. | Typed accessors return concrete `FormatStyle` values callers chain; helpers that need to store "some format" generically use `<F: FormatStyle>` constraints. | Proposed |
| 2026-07-18 | `NebulaMeasureResult` minimal for v1 (name/iterations/total/perIteration); distribution stats deferred | p50/p99 + `Clock.minimumResolution` is a later minor; `bench()` is a quick micro-bench, not statistically rigorous yet. | Minimal `Sendable` struct; min/max/mean/p50/p99 in a later minor release. Document `bench()` as quick, not rigorous. | Proposed |
| 2026-07-18 | `NebulaLogStoreExporter.entries` sync `AnySequence` copying into a Sendable `NebulaLogStoreEntry` (async variant deferred) | `OSLogEntry` is an ObjC class hierarchy (not `Sendable`); crossing actors requires copying out primitives. `OSLogStore.getEntries` returns sync `AnySequence<OSLogEntry>` (Apple also offers async). | Sync wrapper for v1; async variant deferred to a later minor. `NebulaLogStoreEntry` carries composedMessage/date/subsystem/category/level. **Deferred** — `NebulaLogStoreExporter`/`NebulaLogStoreEntry` not shipped in 0.1.0. | Deferred |
| 2026-07-18 | Defer OS 26 signpost-metric APIs to Nebula 27 | `OSMetricOperation` is `@usableFromInline internal` in the os overlay — cannot re-export across module boundaries. | If metric APIs (add/subtract/set/reset) are needed, Nebula defines its own `NebulaMetricOperation` enum gated `@available(iOS 26, *)`; cannot typealias `OSMetricOperation`. Defer to Nebula 27 unless needed now. | Proposed |
| 2026-07-19 | **Nebula Clean Architecture toolkit surface (Wave H)** — ship the seams only, no presentation/DB/framework code | The second surface of Nebula is architecture. Uncle Bob layers mapped to Swift 6 / Nebula constructs; Nebula owns the inner ports/markers/DTO contract + a wiring helper; concrete adapters (repos, gateways, presenters) live in the app. Cosmos is the presentation layer — Nebula defines no presenter. | New `Sources/Nebula/Architecture/` subtree (Domain/Ports/Errors/UseCase/Repository/Gateway/Validation/Registry/Testing/Async). 10 locked design decisions (UseCase granularity fine-grained `NebulaUseCase<I,O>` + `executeTyped`; CQS closed `NebulaUseCaseRole` enum; per-layer open-struct `NebulaFailure` errors bridging to closed `NebulaError.Kind` — NO new cases; capability repository protocols, no CRUD; `NebulaRegistry` DI-without-a-container + explicit-param primary path; sync/async validator split; NO 5th config — decorators route to existing configs; test-doubles in main target; bare `NebulaOutputPort` marker; single-target + documented Domain module). All at the Nebula 26 floor (no above-floor gates); pure Swift + Foundation + `Synchronization`. `Sendable` derived throughout — the toolkit introduces NO `@unchecked Sendable`; the two `final class` test-helpers (`NebulaFakeRepository`/`NebulaSpyUseCase`) derive `Sendable` (final class with all-`let` `Sendable` properties; the spy is explicitly `: Sendable` so it can be shared across tasks), mirroring the `NebulaError.Box` precedent rather than the `NebulaMemoryLogHandler` `@unchecked` exception. Deferred: `NebulaInvariant`, `NebulaMockRepository`, `NebulaHTTPGateway`, `NebulaCancellation`/`NebulaError.wrapAsync`, template multi-module product. Decisions + alternatives recorded in `vault/08-riscos/clean-architecture-open-questions.md`; risks in `vault/08-riscos/clean-architecture-toolkit-risks.md`; canonical surface in `vault/03-padroes/nebula-clean-architecture-toolkit.md`. 509 tests / 107 suites green; zero concurrency warnings. | Accepted |
