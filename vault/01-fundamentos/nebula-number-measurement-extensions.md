It standardizes on Foundation's modern **`FormatStyle`** family ([WWDC21 — What's new in Foundation, Session 10109](https://developer.apple.com/videos/play/wwdc2021/10109/)) and explicitly avoids the legacy `NumberFormatter` / `MeasurementFormatter` / `ByteCountFormatter` classes. The only legacy class Nebula touches is `ListFormatter`, because it has no `FormatStyle` replacement.\n\nThis note is the [[nebula-number-measurement-extensions]] dimension of the [[nebula-spm-architecture]] foundation phase. It must compile on all 5 platforms at `.v26` (iOS 26, macOS 26, tvOS 26, watchOS 26, visionOS 26) under Swift 6 strict concurrency with zero warnings — see [[nebula-swift6-concurrency]].\n\n## Ground truth: FormatStyle availability (verified from the SDK)\n\nEvery claim below is checked against the installed toolchain's `Foundation.swiftinterface` at\n`/Applications/Xcode-27.0.0-Beta.3.app/.../arm64e-apple-macos.swiftinterface` (Xcode 27 Beta 3 / MacOSX 27.0 SDK, 23723 lines) AND cross-checked against the visionOS interface at `.../XROS.platform/.../XROS27.0.sdk/.../arm64e-apple-xros.swiftinterface` (23611 lines). The relevant declarations carry a single, uniform availability marker — there is no per-platform divergence to gate:\n\n```\n@available(macOS 12.0, iOS 15.0, tvOS 15.0, watchOS 8.0, *)\npublic struct IntegerFormatStyle<Value>        : Codable, Hashable, Sendable where Value : BinaryInteger   // line 19881\n@available(macOS 12.0, iOS 15.0, tvOS 15.0, watchOS 8.0, *)\npublic struct FloatingPointFormatStyle<Value>  : Codable, Hashable, Sendable where Value : BinaryFloatingPoint // line 19637\n@available(macOS 12.0, iOS 15.0, tvOS 15.0, watchOS 8.0, *)\npublic struct ByteCountFormatStyle             : FormatStyle, Sendable                                   // line 20433\n@available(macOS 12.0, iOS 15.0, tvOS 15.0, watchOS 8.0, *)\nextension Measurement where UnitType : Dimension { public struct FormatStyle : FormatStyle, Sendable }      // line 16521\n@available(macOS 12.0, iOS 15.0, tvOS 15.0, watchOS 8.0, *)\nextension Decimal { public struct FormatStyle : Sendable }                                                // line 19388\n```\n\nvisionOS verification: the visionOS SDK interface carries the **same** `@available(macOS 12.0, iOS 15.0, tvOS 15.0, watchOS 8.0, *)` markers with visionOS **not named**. Per Swift convention, an omitted platform means the API is available on that platform from its first version — so all of the above are available on **visionOS 1.0+** unconditionally. This was confirmed by reading the visionOS interface directly (FloatingPointFormatStyle at xros-line 19527, IntegerFormatStyle 19771, ByteCountFormatStyle 20323), not merely inferred from iOS availability.\n\nThe `Measurement<UnitInformationStorage>.FormatStyle.ByteCount` variant is one notch newer:\n\n```\n@available(macOS 13.0, iOS 16.0, tvOS 16.0, watchOS 9.0, *)\nextension FormatStyle where Self == Measurement<UnitInformationStorage>.FormatStyle.ByteCount { ... }   // line 16613\n```\n\n`Measurement<UnitType>` itself is iOS 10 / macOS 10.12 / watchOS 3 / tvOS 10 (line 14507), `Sendable` (14518), `Comparable`, `Equatable`, `Codable`, with `converted(to:)` / `convert(to:)` constrained to `UnitType: Dimension` (14534-14535). `MeasurementFormatUnitUsage<UnitType>` is `Codable, Hashable, Sendable` with `.general` / `.asProvided` (line 17296). `Measurement<Unit>.FormatStyle.UnitWidth` is `.wide / .abbreviated / .narrow` and is generic over `UnitType` (16523). `ByteCountFormatStyle.Style` is `Sendable` (20458); `format(_ value: Int64) -> String` (20456).\n\n**Conclusion:** all of these are strictly below Nebula's `.v26` floor (OS 26). On Nebula 26, **no `@available` gating is needed** for this layer, and all 5 platforms are covered unconditionally (visionOS 1+ confirmed via the visionOS SDK interface). Anything above `.v26` (an OS 27 / Nebula 27 API) must be gated — none were found in this dimension.\n\n## Modern vs legacy — what Nebula adopts\n\n| Legacy (ObjC `Formatter` subclass, NOT Sendable) | Modern `FormatStyle` replacement (Sendable, Codable, parseable) |\n| --- | --- |\n| `NumberFormatter` ([docs](https://developer.apple.com/documentation/foundation/numberformatter)) | `IntegerFormatStyle` / `FloatingPointFormatStyle` / `Decimal.FormatStyle`, with `.Percent` / `.currency(code:)` substyles |\n| `MeasurementFormatter` ([docs](https://developer.apple.com/documentation/foundation/measurementformatter)) | `Measurement<Unit>.FormatStyle` via `.measurement(width:usage:numberFormatStyle:)` |\n| `ByteCountFormatter` ([docs](https://developer.apple.com/documentation/foundation/bytecountformatter)) | `ByteCountFormatStyle` (Int64) via `.byteCount(style:allowedUnits:...)`; or `Measurement<UnitInformationStorage>.FormatStyle.ByteCount` (iOS 16+) |\n| `ListFormatter` ([docs](https://developer.apple.com/documentation/foundation/listformatter)) | **No replacement.** Wrapped behind a per-call Sendable facade. |\n\nThe legacy classes are absent as Swift declarations from BOTH the macOS and visionOS `.swiftinterface` files (only `extension Foundation::MeasurementFormatter` at line ~15142 appears) — they live on the Clang module side, confirming they are not modern Swift-first API. Nebula does not store them in `Sendable` state.\n\n> Correction of the dimension brief: `ListFormatter` has **no `ItemStyle` enum** (confirmed via Apple docs). List joining is locale-driven only (`localizedString(byJoining:)`); the `ItemStyle` concept belongs to `DateFormatter.Style` / `NumberFormatter.Style`, not `ListFormatter`. `ListFormatter` availability: iOS 13.0+ / macOS 10.15+ / tvOS 13.0+ / watchOS 6.0+ / visionOS 1.0+.\n\n## Decimal is its own style — not a FloatingPoint\n\n`Decimal` does **not** conform to `FloatingPoint`. The interface marks the FloatingPoint static members and `formTruncatingRemainder` `@available(*, unavailable, message: \"Decimal does not yet fully adopt FloatingPoint.\")` (lines 7075/7079/7150). Therefore:\n\n- Do NOT route `Decimal` through `FloatingPointFormatStyle` — it will not compile.\n- Use `Decimal.FormatStyle` (its own `Sendable` struct, line 19388), with `.number` / `.percent` / `.currency(code:)`.\n- `Decimal` has `formatted<S>(_:) where S.FormatInput == Decimal` (line 19567), so `value.formatted(Decimal.FormatStyle().percent)` compiles.\n\n> **Decimal rounding caveat (verified):** `Decimal` has **no value-level `rounded(_:)` method**. The only `rounded` members in the interface are `Decimal.FormatStyle.rounded(rule:increment:)` and the `Percent` / `Currency` substyle equivalents (lines 19400/19434/19462), all of which return a `FormatStyle` (a style builder), **not** a rounded `Decimal` value. The native rounding primitive is the C function `NSDecimalRound(&result, &number, scale, roundingMode)` with `NSDecimalNumber.RoundingMode` (`.plain` / `.bankers` / `.down` / `.up`), plus manual Decimal multiply/truncate/divide arithmetic. `Decimal.rounded(toDecimalPlaces:)` MUST be implemented via `NSDecimalRound` (or `NSDecimalNumber`) — **not** via a `Decimal` value method. The earlier draft's claim that \"Decimal has a `rounding` API / round via Decimal's `rounded(_:)`\" was wrong and is corrected here.\n\n## Recommended design for Nebula\n\nModule placement (single SPM target `Nebula`, [[nebula-spm-architecture]]):\n- `Sources/Nebula/Extensions/Numbers/` — value-type extensions on `BinaryInteger`, `BinaryFloatingPoint`, `Decimal`.\n- `Sources/Nebula/Formatting/` — `NebulaFormattingOptions`, `NebulaNumberFormatting` facade, `ListFormatter` wrapper.\n\n`import Foundation` only. All public value types get **derived** `Sendable`; the facade is a caseless `enum` (no stored state, implicitly Sendable). No `Mutex`/`Atomic` is needed in this layer because no shared mutable state exists — `ListFormatter` is instantiated per call.\n\n### Public API surface\n\n```swift\n// Formatting/Options/NebulaFormattingOptions.swift\npublic struct NebulaFormattingOptions: Sendable {\n    public var locale: Locale\n    public var percentPrecision: Int?            // -> FloatingPointFormatStyle.Configuration.Precision\n    public var currencyCode: String?             // ISO 4217, e.g. \"USD\"\n    public var byteStyle: ByteCountFormatStyle.Style   // Sendable (interface line 20458)\n    public init(locale: Locale = .autoupdatingCurrent,\n                percentPrecision: Int? = nil,\n                currencyCode: String? = nil,\n                byteStyle: ByteCountFormatStyle.Style = .file)\n    public static let `default` = NebulaFormattingOptions()\n    // Fluent (Cosmos-style, no SwiftUI):\n    public func with(locale: Locale) -> Self\n    public func with(percentPrecision: Int?) -> Self\n    public func with(currencyCode: String?) -> Self\n    public func with(byteStyle: ByteCountFormatStyle.Style) -> Self\n}\n// NOTE: measurementWidth is intentionally NOT a field — Measurement<Unit>.FormatStyle.UnitWidth\n// is generic over Unit: Dimension (interface line 16523) and cannot live in a non-generic\n// Sendable struct without type erasure. `value.formatted(Decimal.FormatStyle().percent)` for Decimal (valid via `Decimal.formatted<S>(_:) where S.FormatInput == Decimal`, line 19567).\n- `currency(_:)` — `value.formatted(.currency(code: options.currencyCode ?? Locale.current.currency?.identifier ?? \"USD\"))`. Prefer the `Decimal` overload for money (no binary-float loss).\n- `bytes(_:)` — `Int64(count).formatted(.byteCount(style: options.byteStyle))` (`ByteCountFormatStyle.format` takes `Int64`, line 20456). No `ByteCountFormatter`.\n- `measurement(_:)` — `m.formatted(.measurement(width: width, usage: usage, numberFormatStyle: nil))` (`Measurement.formatted` requires `UnitType: Dimension`, line 16602). No `MeasurementFormatter`.\n- `list(_:)` — if `options.locale == .autoupdatingCurrent`, call the stateless `ListFormatter.localizedString(byJoining: items)`; otherwise build a local `ListFormatter()`, set `.locale`, call `string(from:)`, discard. The instance is local to the call → no shared mutable state → safe under Swift 6. `ListFormatter` is NOT `Sendable` and MUST NOT be stored in a `Sendable` struct.\n- `BinaryFloatingPoint.rounded(toDecimalPlaces:)` — scale by `pow(10, places)`, round with `rule`, rescale; document Double lossiness; steer money to the `Decimal` overload.\n- **`Decimal.rounded(toDecimalPlaces:)`** — Decimal has NO value-level `rounded(_:)` method (verified). Implement via `NSDecimalRound(&result, &self, places, roundingMode)` with `NSDecimalNumber.RoundingMode` (default `.bankers` to match `.toNearestOrEven` semantics), or manual Decimal multiply/truncate/divide arithmetic. No `Double` round-trip, no `NSDecimalNumber` object allocation required when using `NSDecimalRound`.\n- `measured(in:)` is generic over `Unit: Dimension`, so callers write `100.0.measured(in: UnitLength.kilometers).converted(to: .miles)` — no per-unit property explosion.\n- `NebulaFormattingOptions` deliberately omits `measurementWidth`: it is generic over `Unit: Dimension` (`Measurement<Unit>.FormatStyle.UnitWidth` is generic over `UnitType`, line 16523) and cannot be a non-generic struct field without type erasure. Pass `width` explicitly to `measurement(_:width:options:usage:)` instead (see Open questions).\n\n## Apple patterns adopted\n\n- Modern `FormatStyle` over legacy `Formatter` subclasses — per [WWDC21 Session 10109](https://developer.apple.com/videos/play/wwdc2021/10109/) and corroborated by [WWDC21 notes](https://wwdcnotes.com/notes/wwdc21/10109/).\n- `ParseableFormatStyle` with fluent `rounded(rule:increment:)` / `precision(_:)` / `sign(strategy:)` / `notation(_:)` / `scale(_)` builders instead of mutating a `NumberFormatter`.\n- Static convenience constructors on `FormatStyle` (`.number`, `.percent`, `.currency(code:)`, `.byteCount(...)`, `.measurement(width:usage:)`) used directly inside the facade.\n- `Measurement<Unit>` value type with `converted(to:)` / arithmetic operators for `Dimension`-constrained units; `.measured(in:)` wraps `Measurement(value:unit:)` generically.\n- `Decimal.FormatStyle` as a first-class style (Decimal ≠ FloatingPoint, verified).\n- [[nebula-spm-architecture]] CosmosConfiguration-style contract adapted for a foundation (no SwiftUI): `NebulaFormattingOptions` is a `Sendable` struct + fluent `.with*` builders; the facade is a caseless `enum` (no `@Entry`/`@Observable`).\n- Derived `Sendable` for all public value types; no `@unchecked`, no `Mutex`/`Atomic` in this layer (no shared mutable state).\n- DocC on every public symbol; deprecation via `@available(*, deprecated, message:)` (none planned for v26).\n\n## Risks & open questions\n\n- **ListFormatter not Sendable**: must use the stateless class func or a per-call instance; never cached in a `Sendable` struct. (Mitigated above.)\n- **Two byte-count APIs**: standardize on `ByteCountFormatStyle` (Int64) for `.bytes(_:)`; mention `Measurement<UnitInformationStorage>.FormatStyle.ByteCount` only for callers already holding a `Measurement<UnitInformationStorage>`.\n- **Decimal ≠ FloatingPoint**: routing `Decimal` through `FloatingPointFormatStyle` won't compile — use `Decimal.FormatStyle`.\n- **Decimal has no value-level rounded(_)**: `Decimal.rounded(toDecimalPlaces:)` must use `NSDecimalRound` (C) + `NSDecimalNumber.RoundingMode`, or manual Decimal arithmetic — NOT a Decimal value method (the earlier \"Decimal has a rounding API\" note was wrong).\n- **Double lossiness**: `rounded(toDecimalPlaces:)` on `BinaryFloatingPoint` is lossy; money must use the `Decimal` overload.\n- **`measurementWidth` genericity**: dropped from the shared options struct; passed explicitly. Confirm with user.\n- **watchOS 8 floor**: fine for `.v26`, but document the iOS 15/macOS 12/tvOS 15/watchOS 8 / visionOS 1+ minimum for this layer in case of future back-deployment.\n- **Open**: expose `AttributedString`-producing variants (`.attributed` is iOS 15+; `ByteCountFormatStyle.attributed` and `Decimal.FormatStyle.Attributed` confirmed present) for [[nebula-string-extensions]] / Cosmos integration, or keep String-only for v1?\n- **Open**: `.measured(in:)` generic builder only, or also a curated set of unit-literal extensions (`100.kilometers`)? Recommend generic only.\n- **Open**: one shared `NebulaFormattingOptions` vs per-domain (`NebulaPercentOptions` / `NebulaCurrencyOptions` / `NebulaByteOptions` / `NebulaMeasurementOptions<Unit>`). Recommend shared for v1.\n- **Open**: `Decimal.rounded(toDecimalPlaces:)` rule type — Swift `FloatingPointRoundingRule` (mapped to `NSDecimalNumber.RoundingMode`), a `NebulaDecimalRoundingRule` enum mirroring `NSDecimalNumber.RoundingMode`, or fixed `.toNearestOrEven` (bankers) for v1? Recommend a small `NebulaDecimalRoundingRule` enum mapping 1:1 to `NSDecimalNumber.RoundingMode`.\n\n## Sources\n\n- [What's new in Foundation — WWDC21 Session 10109](https://developer.apple.com/videos/play/wwdc2021/10109/)\n- [WWDC21 notes — What's new in Foundation](https://wwdcnotes.com/notes/wwdc21/10109/)\n- [Foundation.swiftinterface — macOS (local, Xcode 27 Beta 3 / MacOSX 27.0 SDK)](file:///Applications/Xcode-27.0.0-Beta.3.app/Contents/Developer/Platforms/MacOSX.platform/Developer/SDKs/MacOSX27.0.sdk/System/Library/Frameworks/Foundation.framework/Versions/C/Modules/Foundation.swiftmodule/arm64e-apple-macos.swiftinterface)\n- [Foundation.swiftinterface — visionOS (local, Xcode 27 Beta 3 / XROS 27.0 SDK)](file:///Applications/Xcode-27.0.0-Beta.3.app/Contents/Developer/Platforms/XROS.platform/Developer/SDKs/XROS27.0.sdk/System/Library/Frameworks/Foundation.framework/Modules/Foundation.swiftmodule/arm64e-apple-xros.swiftinterface)\n- [ListFormatter — Apple Developer Documentation](https://developer.apple.com/documentation/foundation/listformatter)\n- [localizedString(byJoining:) — Apple Developer Documentation](https://developer.apple.com/documentation/foundation/listformatter/localizedstring(byjoining:))\n- [NumberFormatter — Apple Developer Documentation](https://developer.apple.com/documentation/foundation/numberformatter)\n- [MeasurementFormatter — Apple Developer Documentation](https://developer.apple.com/documentation/foundation/measurementformatter)\n- [ByteCountFormatter — Apple Developer Documentation](https://developer.apple.com/documentation/foundation/bytecountformatter)\n- [NSHipster — Formatter (2019)](https://github.com/NSHipster/articles/blob/master/2019-07-15-formatter.md)\n- [Hacking with Swift — How to join an array of strings in a natural way](https://www.hackingwithswift.com/example-code/system/how-to-join-an-array-of-strings-in-a-natural-way)\n\nSource of truth = root docs (CLAUDE.md, ARCHITECTURE.md, DECISIONS.md, VERSIONING.md);
