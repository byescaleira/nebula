All ground truth below is from the installed `Foundation.swiftinterface` (MacOSX27.0.sdk) and Apple docs / SE-0329 for the `_Concurrency` stdlib types (which ship no textual `.swiftinterface` in the Xcode 27 Beta.3 toolchain — only binary prebuilt `.swiftmodule`).\n\n## Core types and their availability\n\n| Type | Floor (iOS / macOS / tvOS / watchOS / visionOS) | Sendable | Notes |\n|---|---|---|---|\n| `Date` | universal | yes | `Comparable, Hashable, Sendable, Codable`; `Strideable` iOS 16/macOS 13/tvOS 16/watchOS 9; `Date.now` back-deployed iOS 15/macOS 12/tvOS 15/watchOS 8 ([Date](https://developer.apple.com/documentation/foundation/date)) |\n| `DateComponents` | iOS 8 / macOS 10.9 / tvOS 9 / watchOS 2 | yes | `DateComponents.formatted<F>` and `DateComponents.ISO8601FormatStyle` are **NEW at OS 26** ([DateComponents](https://developer.apple.com/documentation/foundation/datecomponents)) |\n| `DateInterval` | iOS 10 / macOS 10.12 / tvOS 10 / watchOS 3 | yes (Codable) | `Comparable, Hashable, Codable, Sendable`; already has `init(start:duration: TimeInterval)` |\n| `Calendar`, `TimeZone` | universal | yes | Thread a `Calendar` through every arithmetic helper. Nebula needs **no `@available` gates** in this layer; everything compiles on iOS/macOS/tvOS/watchOS/visionOS `.v26`.\n\n## Corrections verified against the SDK\n\n1. **There is no `.offset` / `.timer` in Foundation.** The research brief listed `Date.RelativeFormatStyle (.relative/.offset/.timer)`. Verified: Foundation provides only `public static func relative(presentation:unitsStyle:) -> Self` (interface line 18434). There is **no `static var relative`** (`grep -nc 'static var relative'` = 0) and **no `.offset` or `.timer`** member anywhere. The timer-style display is `Duration.TimeFormatStyle.Pattern.hourMinuteSecond` (renders `HH:mm:ss`). Nebula must not expose `.offset`/`.timer` as Foundation-backed API.\n\n2. **There is no `Calendar.gregorian` static.** Foundation exposes only `Calendar.Identifier.gregorian` (enum case, line 1943) and `Calendar(identifier:)` (init, line 2028). Foundation's own code uses `Calendar(identifier: .gregorian)` (line 19323). The `verbatim` preset's default `calendar: Calendar = .gregorian` would NOT compile — use `Calendar(identifier: .gregorian)`.\n\n3. **RelativeFormatStyle is not fluent.** `presentation`/`unitsStyle` are stored `var` properties, not fluent setter methods returning `Self`. Construct via `Date.RelativeFormatStyle(presentation: .numeric, unitsStyle: .narrow)` (init, line 18411). Only `Date.FormatStyle` (`.dateTime.day().month().year()`) is genuinely fluent.\n\n4. **`DateInterval.init(start:duration:)` already exists** in Foundation taking `TimeInterval` (line 13718). A Nebula bridge must be a **distinct overload taking `Swift.Duration`**, explicitly typed to avoid ambiguity.\n\n5. **`DateComponents.formatted` / `DateComponents.ISO8601FormatStyle` are NEW at OS 26** (lines 2201/8344, including visionOS 26). Usable ungated at the v26 floor, but if the deployment target is ever lowered they need `@available(iOS 26, *)` gating.\n\n## Why FormatStyle, not DateFormatter\n\n`DateFormatter`, `ISO8601DateFormatter`, `RelativeDateTimeFormatter`, and `DateComponentsFormatter` are legacy `NSObject` subclasses — not `Sendable`, thread-unsafe, and mutable. The modern path (Apple's recommendation since iOS 15/macOS 12) is the `FormatStyle` family: value types, `Sendable`, `Codable`, `Hashable`, and `ParseableFormatStyle` (symmetric format + parse via `Date(_:strategy:)`, lines 2195-2199). See [[nebula-codable-foundation]] for parse interplay.\n\n## Recommended design for Nebula\n\nNebula does **not** build a new formatting engine. It (a) extends the Foundation value types with calendar-aware helpers and (b) exposes a thin `Nebula*` formatting facade of pre-configured `FormatStyle` constants/functions.\n\n### Module placement The facade's verbatim preset must use the latter.\n- `.offset`/`.timer` are **not** Foundation API (only `.relative(presentation:unitsStyle:)`). Do not invent them.\n- `DateComponents.formatted` / `DateComponents.ISO8601FormatStyle` are NEW at OS 26 — usable ungated at the v26 floor, but if the deployment target is ever lowered they need `@available(iOS 26, *)` gating.\n- `Date.AnchoredRelativeFormatStyle`, `*.Attributed`, `DiscreteFormatStyle`, `allowedFields` are iOS 18/macOS 15/tvOS 18/watchOS 11 — below v26, fine.\n- `RelativeFormatStyle` is **not** fluent — use the initializer; only `Date.FormatStyle` is fluent.\n- FormatStyle defaults use `.autoupdatingCurrent` — output drifts with user settings. For logs/persistence, pin `Locale(identifier: \"en_US_POSIX\")` + `.gmt`; keep a `.localized` variant for UI.\n- `Duration` is `Swift.Duration` (`_Concurrency`), not a `Foundation.Duration` distinct type. Converting to `DateInterval` needs an explicit, distinctly-typed `Swift.Duration` overload (Foundation already has the `TimeInterval` initializer).\n- `_Concurrency` Clock/Instant have **no** textual `.swiftinterface` (only binary `.swiftmodule`) — ground truth rests on Apple docs + SE-0329 + open-source Swift stdlib. Availability (iOS 16/macOS 13/tvOS 16/watchOS 9/visionOS 1) is below the v26 floor; confirm at compile time.\n- DST: compute `endOfDay` via `calendar.dateInterval(of:.day, for:)` (Foundation handles 23/25-hour days), never `startOfDay + 86_400`.\n- The Apple doc URL `…/foundation/duration` currently 404s; update the source link once the canonical slug is known.\n\n### Open questions\n- Extend `Date` directly vs. a `NebulaDate` wrapper? Recommend extend directly (interop, no boxing). Confirm with user.\n- `static let` vs `static func` presets? Recommend constants for fixed presets, funcs for parameterized. Confirm.\n- Boundary between `NebulaClock` here and signpost measurement in [[nebula-standardize-measure]]? Recommend primitive timing here, signposts there. Confirm.\n- Pin ISO/stable presets to `en_US_POSIX` + `.gmt`? Recommend yes for logs/persistence; `.localized` for UI. Confirm.\n- `endOfDay` semantics: inclusive last instant vs exclusive start-of-next-day? Recommend inclusive-last-instant, documented.\n- Confirm the canonical Apple doc URL for `Duration` (current slug 404s).\n\n## Sources\n\n- [Date — Foundation | Apple Developer
