# Nebula ‚Äî Knowledge Vault

Synthesis/navigation layer for the Nebula foundation/architecture SwiftPM library (sibling of Cosmos, the SwiftUI design system). The **source of truth stays the root docs** (`ARCHITECTURE.md`, `DECISIONS.md`, `VERSIONING.md`, `CLAUDE.md`, `PROPOSAL.md`, `ROADMAP.md`, `CONTRIBUTING.md`); this vault is a synthesis layer. On conflict, the root doc wins ‚Äî update the note.

The `.swiftinterface` (Xcode 27 Beta 3 SDK) is the authoritative ground truth for Apple API availability ‚Äî not WebFetch, which has hallucinated availability tables repeatedly (UUID.random, percentEncodedQueryItems, `OutputFormatting.fragmentsAllowed`, `convertFromKebabCase`, `base64Encode`, a parameterless `UUID.random()`, "UUID is not Comparable"). Cite interface line numbers when verifying.

- [[nebula-error-taxonomy-toolkit]] ‚Äî architecture-toolkit error dimension: `NebulaFailure: Error, Sendable` protocol + per-layer open structs (`NebulaDomainError`/`NebulaRepositoryError`/`NebulaValidationError`) bridging to the CLOSED `NebulaError.Kind` enum via caller-picked `toNebulaError(kind:)` ‚Äî NO new Kind cases; fine taxonomy in open `var code: String`. - [[nebula-swift6-concurrency]] ‚Äî Swift 6 concurrency foundation: Sendable-by-derivation; `@Sendable` handlers; no `NSLock`/`DispatchQueue`/`nonisolated(unsafe)` ‚Üí `Mutex<T>`/`Atomic<T>` from `import Synchronization` (SwiftStdlib 6.0 ‚Üí iOS 18/macOS 15, below `.v26`); once-token `static let` + `swift_once` for idempotent setup; region-based isolation (SE-0414) before `@unchecked`; typed throws (SE-0413) opt-in with `NebulaError` as concrete `Failure`; no SwiftUI constructs (`@Entry`/`@Observable`/`@Environment`); corrected `Atomic` ordering types (three frozen structs ‚Äî `AtomicLoadOrdering`/`AtomicStoreOrdering`/`AtomicUpdateOrdering`, NOT a single `Ordering` enum; `.acquiringAndReleasing` invalid for `load`); `Mutex.withLock` uses `sending` (SE-0430) not `transferring`; `NebulaLocked`/`NebulaFlag`/`NebulaOnce` `~Copyable` wrappers.

## Risks index

- [[above-floor-26-4-apis]] ‚Äî above-floor (Nebula 26.4) APIs requiring `@available(iOS 26.4, macOS 26.4, tvOS 26.4, watchOS 26.4, visionOS 26.4, *)` gates: `Data.Base64EncodingOptions.base64URLAlphabet`/`.omitPaddingCharacter`, `String.Encoding.ianaName` getter AND `init?(ianaName:)`, `UUID.random(using:)` (swift-foundation SF-0031 / FoundationPreview 6.3, surfaced as OS 26.4; NOT parameterless `random()`).
- [[osmetricoperation-internal]] ‚Äî `OSMetricOperation` (the OS 26 signpost-metrics enum) is `@usableFromInline internal` in the os overlay ‚Äî cannot be re-exported across module boundaries. If NebulaSignposter exposes metric APIs, Nebula must define its own public `NebulaMetricOperation` enum gated `@available(iOS 26, *)`; defer to Nebula 27.
- [[visionos-oslogstore-scope-uncertain]] ‚Äî visionOS availability of `OSLogStore.Scope.system` and `.local()` is UNCERTAIN (the `Store.h` header does NOT list visionOS in `API_UNAVAILABLE`). Treat as macOS-only by safe gating (`#if os(macOS)` or explicit per-platform `@available(<platform>, unavailable)` including visionOS); confirm at compile time on the visionOS SDK.
- [[webfetch-availability-hallucinations]] ‚Äî WebFetch on `developer.apple.com` has repeatedly hallucinated availability tables (UUID.random iOS 13+ + parameterless `random()` + "UUID is not Comparable"; `percentEncodedQueryItems` iOS 16+; `OutputFormatting.fragmentsAllowed`; `convertFromKebabCase`; `base64Encode`). The `.swiftinterface` (Xcode 27 Beta 3 SDK) is authoritative; never rely on WebFetch for availability.
- [[mutex-sending-not-transferring]] ‚Äî `Mutex.withLock` uses `sending` (SE-0430 standardized on `sending`), NOT `transferring` (the earlier SE-0433 spelling was revised). Wrapper forwarding signatures must use `sending` to match.
- [[atomic-ordering-three-structs]] ‚Äî there is NO single `Ordering` enum. Three frozen Sendable structs: `AtomicLoadOrdering` (`.relaxed`/`.acquiring`/`.sequentiallyConsistent` ‚Äî `.acquiringAndReleasing` is INVALID for `load`), `AtomicStoreOrdering`, `AtomicUpdateOrdering` (`.acquiringAndReleasing` valid here, used by `compareExchange`/`exchange`). `NebulaFlag.load` uses `.acquiring`; `set` uses `.releasing`; `compareExchange` uses `.acquiringAndReleasing`.
