# Nebula — Knowledge Vault

Synthesis/navigation layer for the Nebula foundation/architecture SwiftPM library (sibling of Cosmos, the SwiftUI design system). The **source of truth stays the root docs** (`ARCHITECTURE.md`, `DECISIONS.md`, `VERSIONING.md`, `CLAUDE.md`, `PROPOSAL.md`, `ROADMAP.md`, `CONTRIBUTING.md`); this vault is a synthesis layer. On conflict, the root doc wins — update the note.

The `.swiftinterface` (Xcode 27 Beta 3 SDK) is the authoritative ground truth for Apple API availability — not WebFetch, which has hallucinated availability tables repeatedly (UUID.random, percentEncodedQueryItems, `OutputFormatting.fragmentsAllowed`, `convertFromKebabCase`, `base64Encode`, a parameterless `UUID.random()`, "UUID is not Comparable"). Cite interface line numbers when verifying.

## Patterns index

- [[nebula-spm-architecture]] — single-target SPM manifest mirroring Cosmos: `swift-tools-version: 6.3` (dual Xcode 26.4+ / Xcode 27 build; Xcode 26.0–26.3 shipped Swift 6.2 and will NOT parse a 6.3 manifest — require Xcode 26.4+), 5 platforms `.v26`, `swiftLanguageModes: [.v6]`, `defaultLocalization: "en"`, single `.library` product `Nebula` + `.testTarget` `NebulaTests` (Swift Testing), no third-party deps, `.process("Resources")` commented (no String Catalog by default), DocC catalog auto-discovered at `Sources/Nebula/Nebula.docc/`, OS-27-only SDK symbols compile-gated `#if swift(>=6.4)`, platform-specific surfaces `#if os(...)`.
- [[nebula-clean-architecture]] — Uncle Bob Clean Architecture layers → Swift 6 / Nebula constructs (Entity `Sendable` struct/`actor`; UseCase `Sendable` protocol with `@Sendable execute` + `Input/Output: Sendable`, untyped `throws`; Interface Adapters = `NebulaInputPort`/`NebulaOutputPort`/`NebulaRepository`/`NebulaGateway` Sendable protocols owned inner, implemented outer; `NebulaDTO` marker; Frameworks & Drivers outside Nebula). Dependency rule enforced by protocol ownership + a lightweight `NebulaCompositionRoot` (Mutex-backed factory bindings, NOT a DI container; `dependencies: []` forbids Resolver/Factory/Swinject). DIP = inner port defined in Nebula, outer presenter/repo implements. DTOs cross boundaries (never Entities). Fowler Repository/Gateway. Single-target → compile-time enforcement needs a separate app module (open risk). Out of scope: MVVM/MVC/VIP/VIPER.
- [[nebula-error-taxonomy-toolkit]] — architecture-toolkit error dimension: `NebulaFailure: Error, Sendable` protocol + per-layer open structs (`NebulaDomainError`/`NebulaRepositoryError`/`NebulaValidationError`) bridging to the CLOSED `NebulaError.Kind` enum via caller-picked `toNebulaError(kind:)` — NO new Kind cases; fine taxonomy in open `var code: String`. SE-0413 typed throws per layer (`throws(NebulaDomainError)` etc.), untyped `throws`/`Result<T, NebulaError>` at boundaries. Rejects single-`NebulaDomainError` (no layer enforcement).
- [[nebula-swift6-concurrency]] — Swift 6 concurrency foundation: Sendable-by-derivation; `@Sendable` handlers; no `NSLock`/`DispatchQueue`/`nonisolated(unsafe)` → `Mutex<T>`/`Atomic<T>` from `import Synchronization` (SwiftStdlib 6.0 → iOS 18/macOS 15, below `.v26`); once-token `static let` + `swift_once` for idempotent setup; region-based isolation (SE-0414) before `@unchecked`; typed throws (SE-0413) opt-in with `NebulaError` as concrete `Failure`; no SwiftUI constructs (`@Entry`/`@Observable`/`@Environment`); corrected `Atomic` ordering types (three frozen structs — `AtomicLoadOrdering`/`AtomicStoreOrdering`/`AtomicUpdateOrdering`, NOT a single `Ordering` enum; `.acquiringAndReleasing` invalid for `load`); `Mutex.withLock` uses `sending` (SE-0430) not `transferring`; `NebulaLocked`/`NebulaFlag`/`NebulaOnce` `~Copyable` wrappers.

## Risks index

- [[above-floor-26-4-apis]] — above-floor (Nebula 26.4) APIs requiring `@available(iOS 26.4, macOS 26.4, tvOS 26.4, watchOS 26.4, visionOS 26.4, *)` gates: `Data.Base64EncodingOptions.base64URLAlphabet`/`.omitPaddingCharacter`, `String.Encoding.ianaName` getter AND `init?(ianaName:)`, `UUID.random(using:)` (swift-foundation SF-0031 / FoundationPreview 6.3, surfaced as OS 26.4; NOT parameterless `random()`).
- [[osmetricoperation-internal]] — `OSMetricOperation` (the OS 26 signpost-metrics enum) is `@usableFromInline internal` in the os overlay — cannot be re-exported across module boundaries. If NebulaSignposter exposes metric APIs, Nebula must define its own public `NebulaMetricOperation` enum gated `@available(iOS 26, *)`; defer to Nebula 27.
- [[visionos-oslogstore-scope-uncertain]] — visionOS availability of `OSLogStore.Scope.system` and `.local()` is UNCERTAIN (the `Store.h` header does NOT list visionOS in `API_UNAVAILABLE`). Treat as macOS-only by safe gating (`#if os(macOS)` or explicit per-platform `@available(<platform>, unavailable)` including visionOS); confirm at compile time on the visionOS SDK.
- [[webfetch-availability-hallucinations]] — WebFetch on `developer.apple.com` has repeatedly hallucinated availability tables (UUID.random iOS 13+ + parameterless `random()` + "UUID is not Comparable"; `percentEncodedQueryItems` iOS 16+; `OutputFormatting.fragmentsAllowed`; `convertFromKebabCase`; `base64Encode`). The `.swiftinterface` (Xcode 27 Beta 3 SDK) is authoritative; never rely on WebFetch for availability.
- [[mutex-sending-not-transferring]] — `Mutex.withLock` uses `sending` (SE-0430 standardized on `sending`), NOT `transferring` (the earlier SE-0433 spelling was revised). Wrapper forwarding signatures must use `sending` to match.
- [[atomic-ordering-three-structs]] — there is NO single `Ordering` enum. Three frozen Sendable structs: `AtomicLoadOrdering` (`.relaxed`/`.acquiring`/`.sequentiallyConsistent` — `.acquiringAndReleasing` is INVALID for `load`), `AtomicStoreOrdering`, `AtomicUpdateOrdering` (`.acquiringAndReleasing` valid here, used by `compareExchange`/`exchange`). `NebulaFlag.load` uses `.acquiring`; `set` uses `.releasing`; `compareExchange` uses `.acquiringAndReleasing`.

## Data + Network architecture index (the fourth surface — researched 2026-07-19; Q1 open)

The next surface: **data (SwiftData, UserDefaults) + network (remote)**. SwiftData facts verified by `swiftc -typecheck` probe (NOT WebFetch): `ModelContainer`/`FetchDescriptor`/`PersistentIdentifier` are `Sendable`; `ModelContext` and `@Model` classes are **not** (`@available(*, unavailable)` on the conformance) → SwiftData's concurrency model is `@ModelActor`, which maps onto Nebula's async `NebulaRepository` ports (already shipped in Wave H — no new port needed). Network (`URLSession`) and `UserDefaults` are Foundation → ship in Nebula; SwiftData is a 6th framework → binding-rule placement decision (the Q4 of this cycle). Recommended waves N1–N4 (network → UserDefaults → SwiftData → governance, tag `0.4.0`).
