"---\ntags: [foundation, swift6-concurrency]\naliases: [nebula-concurrency, nebula-swift6]\nrelated: [[nebula-spm-architecture]], [[nebula-logging]], [[nebula-errors]]\n---\n\n# Nebula Swift 6 Concurrency Foundation\n\nStrict-concurrency rules and a `Core/Concurrency` helpers module for **Nebula** — the codename sibling of [[nebula-spm-architecture|Cosmos]], but a non-SwiftUI **foundation** package (logging, errors, extensions, formatting, measure). The binding reference is the Cosmos `CLAUDE.md` concurrency section and `DECISIONS.md` ADRs ([[nebula-spm-architecture]]); this note adapts the Sendable-struct + `@Sendable`-handler + `.with*` builder pattern WITHOUT SwiftUI `@Entry`/`@Observable`/`@Environment`.\n\n## Ground truth — what the installed toolchain actually ships\n\nXcode 27 Beta 3 / Swift 6.4 ships the `Synchronization` and `_Concurrency` stdlib modules **only as prebuilt binary `.swiftmodule`** at `usr/lib/swift/<platform>/prebuilt-modules/27.0/{Synchronization,_Concurrency}.swiftmodule/<arch>` — no textual `.swiftinterface` for these modules exists in the toolchain (verified: `find ... -iname '*.swiftinterface'` for Synchronization/_Concurrency returns nothing; only Cxx/Playgrounds/LiveExecutionResultsLogger have textual interfaces). Per the binding constraint's fallback rule, ground truth comes from [Apple's Synchronization docs](https://developer.apple.com/documentation/synchronization), the [open-source Swift stdlib](https://github.com/swiftlang/swift/blob/main/stdlib/public/Synchronization/Mutex/Mutex.swift), Swift Evolution, and the binary module's symbol manglings.\n\n> Verification note (2026-07-18): the binary `.swiftmodule` manglings were inspected directly to confirm method signatures and ordering-type parameter types. Apple's `/documentation/synchronization` page did not render its availability table via fetch (title only); the per-platform numbers below come from the well-established SwiftStdlib-6.0↔OS mapping corroborated by the module's presence in the Xcode 27 Beta 3 toolchain targeting `arm64-apple-macosx26.0`.\n\n### `Mutex` (SE-0433, Swift 6.0)\n\nFrom [stdlib/public/Synchronization/Mutex/Mutex.swift](https://github.com/swiftlang/swift/blob/main/stdlib/public/Synchronization/Mutex/Mutex.swift):\n\n```swift\n@available(SwiftStdlib 6.0, *)\n@frozen\n@_staticExclusiveOnly\npublic struct Mutex<Value: ~Copyable>: ~Copyable {\n  public init(_ initialValue: consuming sending Value)\n  public borrowing func withLock<R, E>(_ body: (inout sending Value) throws(E) -> sending R) throws(E) -> sending R\n  public borrowing func withLockIfAvailable<R, E>(_ body: (inout sending Value) throws(E) -> sending R) throws(E) -> sending R?\n}\nextension Mutex: @unchecked Sendable where Value: ~Copyable {}   // effectively unconditional\n```\n\n- The keyword is **`sending`**, not `transferring` (SE-0430 standardized on `sending`; SE-0433's earlier `transferring` spelling was revised). Verified against the stdlib source.\n- `@_staticExclusiveOnly` → must be declared `let`, never `var`.\n- `@unchecked Sendable` is **justified** here: the lock guarantees exclusive access; the compiler can't see through `os_unfair_lock` (Darwin backend — confirmed by the `_MutexHandle` mangling `So16os_unfair_lock_sV`), `futex` (Linux), `SRWLOCK` (Windows). Canonical Apple-sanctioned `@unchecked` — [SE-0433](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0433-mutex.md).\n- No bare `lock()`/`unlock()` in the safe API — closure-based `withLock`/`withLockIfAvailable` keep both calls in one borrow (avoids async-suspension-between-lock-and-unlock bugs). Unsafe `_unsafeLock`/`_unsafeUnlock` exist only on `where Value == Void` and `@unsafe`-annotated handle wrappers.\n\n### `Atomic` (Swift 6.0)\n\nFrom [stdlib/public/Synchronization/Atomics/Atomic.swift](https://github.com/swiftlang/swift/blob/main/stdlib/public/Synchronization/Atomics/Atomic.swift) and [AtomicMemoryOrderings.swift](https://github.com/swiftlang/swift/blob/main/stdlib/public/Synchronization/Atomics/AtomicMemoryOrderings.swift):\n\n```swift\n@available(SwiftStdlib 6.0, *)\n@frozen\n@_rawLayout(like: Value.AtomicRepresentation)\n@_staticExclusiveOnly\npublic struct Atomic<Value: AtomicRepresentable>: ~Copyable {\n  public init(_ value: consuming Value)\n  public func load(ordering: AtomicLoadOrdering = .sequentiallyConsistent) -> Value\n  public func store(_ desired: Value, ordering: AtomicStoreOrdering = .sequentiallyConsistent)\n  public func exchange(_ desired: Value, ordering: AtomicUpdateOrdering = .sequentiallyConsistent) -> Value\n  public func compareExchange(expected:desired:ordering:) -> (exchanged: Bool, original: Value)   // single-ordering overload (AtomicUpdateOrdering)\n  public func compareExchange(expected:desired:successOrdering:failureOrdering:) -> (exchanged: Bool, original: Value)\n  public func weakCompareExchange(expected:desired:ordering:) -> (exchanged: Bool, original: Value)\n  public func weakCompareExchange(expected:desired:successOrdering:failureOrdering:) -> (exchanged: Bool, original: Value)\n}\nextension Atomic: @unchecked Sendable where Value: Sendable {}\n```\n\n**Ordering types — IMPORTANT (corrected).** There is **no single `Ordering` enum**. There are three **frozen Sendable structs**: `AtomicLoadOrdering`, `AtomicStoreOrdering`, `AtomicUpdateOrdering`, each `@available(SwiftStdlib 6.0, *) @frozen`. Their cases (static vars):\n\n| Struct | Cases |\n|---|---|\n| `AtomicLoadOrdering` | `.relaxed`, `.acquiring`, `.sequentiallyConsistent` |\n| `AtomicStoreOrdering` | `.relaxed`, `.releasing`, `.sequentiallyConsistent` |\n| `AtomicUpdateOrdering` | `.relaxed`, `.acquiring`, `.releasing`, `.acquiringAndReleasing`, `.sequentiallyConsistent` |\n\n`.acquiringAndReleasing` exists **only** in `AtomicUpdateOrdering`. **`load(ordering:)` takes `AtomicLoadOrdering`** — passing `.acquiringAndReleasing` to `load` will NOT compile (confirmed by binary mangling `Atomic...4load8orderingxAA0B12LoadOrderingV_tF`). `store` takes `AtomicStoreOrdering`; `compareExchange`/`exchange` take `AtomicUpdateOrdering`. Verified via binary-module manglings (`4load8orderingx...LoadOrdering`, `5store_8orderingyxn_...StoreOrdering`, `15compareExchange8expected7desired8ordering...UpdateOrdering`).\n\n`Value: AtomicRepresentable` — only primitives (`Bool`, `Int*`, `RawPointer`); arbitrary types need `Mutex`. `Bool: AtomicRepresentable` is provided by `AtomicBool.swift`.\n\n### Per-platform availability\n\n| API | iOS | macOS | tvOS | watchOS | visionOS | Swift |\n|---|---|---|---|---|---|---|\n| `Mutex` / `Atomic` / ordering structs | 18.0+ | 15.0+ | 18.0+ | 11.0+ | 2.0+ | 6.0+ |\n| `Clock` / `ContinuousClock` / `SuspendingClock` / `Duration` | 16.0+ | 13.0+ | 16.0+ | 9.0+ | 1.0+ | 5.7+ |\n\nSource: Synchronization framework [Apple docs](https://developer.apple.com/documentation/synchronization) + [Clock docs](https://developer.apple.com/documentation/swift/clock); SwiftStdlib-6.0↔OS mapping corroborated by the module's presence in the Xcode 27 Beta 3 toolchain.\n\n**Every row is below Nebula's `.v26` floor** (iOS 26 / macOS 26 / tvOS 26 / watchOS 26 / visionOS 26). Conclusion: **no `@available` gating is needed in Nebula for any of these primitives**. The single above-floor item is [SE-0470 isolated conformances](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0470-isolated-conformances.md) (Swift 6.2, gated behind experimental feature flags `IsolatedConformances`/`StrictSendableMetatypes`) — flagged, do not rely on in Nebula 26; revisit for Nebula 27.\n\n## Binding concurrency rules (Nebula-wide)\n\nMirrors the Cosmos `CLAUDE.md` \"Modern Swift 6 concurrency — zero warnings\" section and `DECISIONS.md` ADR \"no NSLock\", adapted for a foundation.\n\n1. **All public value types `Sendable` via derived conformance** ([SE-0302](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0302-concurrent-value-and-concurrent-closures.md)) — declare `: Sendable`, let the compiler synthesize/verify. Never author `@unchecked Sendable` on a Nebula-defined type.\n2. **`@unchecked Sendable` ONLY** for thin wrappers around `Mutex`/`Atomic`-protected state or Apple-framework bridges, with an inline `// @unchecked because:` comment. (Nebula itself does not ship such a public wrapper — `NebulaLocked` below is `Sendable` by delegating to `Mutex`, not by `@unchecked`.)\n3. **`@Sendable` handler closures** in every configuration contract — [[nebula-logging|NebulaLogConfiguration]], [[nebula-errors|NebulaErrorConfiguration]], [[nebula-standardize-measure|NebulaStandards]], [[nebula-standardize-measure|NebulaMeasure]]. Default handler is `{ _ in }` (capture-free, trivially Sendable).\n4. **No `NSLock`, no `DispatchQueue` for synchronization, no `nonisolated(unsafe)` mutable globals.** Shared mutable state uses `Mutex<T>` / `Atomic<T>` from `import Synchronization`.\n5. **Mutex/Atomic are `~Copyable` + `@_staticExclusiveOnly`** → always `let`. A `Mutex`-typed stored property makes the owning type `~Copyable`; prefer keeping shared-state containers as standalone `let` globals or inside `actor`-isolated types.\n6. **Once-token pattern** for idempotent setup: a `static let` whose initializer side-effect runs exactly once via [`swift_once`](https://github.com/swiftlang/swift/blob/main/stdlib/public/runtime/Once.cpp) (Darwin backend: `dispatch_once_f`). No lock primitive. Side-effect-only form: `private static let _register: Void = { NebulaStandards.bootstrap() }()`. ([Once.h](https://github.com/apple/swift/blob/main/include/swift/Runtime/Once.h).)\n7. **[Region-based isolation (SE-0414)](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0414-region-based-isolation.md) is the default alternative to `@unchecked`** — restructure so the compiler proves the non-Sendable transfer is safe (value unused after the boundary) before reaching for the escape hatch.\n8. **Actors, not global actors.** Nebula is non-SwiftUI → no `@MainActor` default isolation. Use `actor` ([SE-0306](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0306-actors.md)) only when shared mutable state spans many call sites and a single `Mutex` is awkward. Avoid `@globalActor` ([SE-0316](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0316-global-actors.md)) in a foundation; the consuming app supplies isolation.\n9. **Typed throws ([SE-0413](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0413-typed-throws.md)).** `NebulaError` is a concrete `Sendable: Error`. APIs that fail with a single known error declare `throws(NebulaError)`; generic APIs use untyped `throws` (== `throws(any Error)`); infallible APIs are `throws(Never)`. Error types crossing actor boundaries MUST be `Sendable` — derive it. SE-0413 imposes no `Sendable` requirement on the thrown type itself, but the isolation boundary does.\n10. **No SwiftUI concurrency constructs.** Nebula uses none of `@Entry`/`@Observable`/`@Environment`/`@MainActor`-default. Configurations are `Sendable` structs + `@Sendable` handlers + fluent `.with*` builders; the app injects them explicitly.\n\n## Recommended design for Nebula\n\n### Module placement\n\nSingle SPM target `Nebula` ([[nebula-spm-architecture]]). Concurrency helpers live in `Sources/Nebula/Core/Concurrency/`. Rules apply across all subfolders (Core/Logging/Errors/Extensions/Formatting/Measure). One `import Nebula` for consumers; Nebula internally `import Synchronization`, `import _Concurrency`.\n\n### Core/Concurrency helpers\n\n```swift\nimport Synchronization\nimport _Concurrency\n\n/// Sendable ~Copyable wrapper around `Mutex<Value>` for shared mutable state.\n/// Delegates to `Mutex`; `Sendable` by delegation (Mutex is unconditionally @unchecked Sendable, SE-0433).\n/// Always held as `let` (Mutex is @_staticExclusiveOnly).\npublic struct NebulaLocked<Value: ~Copyable & Sendable>: ~Copyable, Sendable {\n    @usableFromInline internal let _mutex: Mutex<Value>\n    public init(_ value: consuming sending Value) { _mutex = Mutex(value) }\n    public borrowing func withLock<R: ~Copyable, E: Error>(\n        _ body: (inout sending Value) throws(E) -> sending R\n    ) throws(E) -> sending R { try _mutex.withLock(body) }\n    public borrowing func withLockIfAvailable<R: ~Copyable, E: Error>(\n        _ body: (inout sending Value) throws(E) -> sending R\n    ) throws(E) -> sending R? { try _mutex.withLockIfAvailable(body) }\n}\n\n/// Sendable ~Copyable wrapper around `Atomic<Bool>` for a single thread-safe flag.\n/// NOTE: load takes AtomicLoadOrdering (.relaxed/.acquiring/.sequentiallyConsistent) —\n/// NOT .acquiringAndReleasing (that case lives only in AtomicUpdateOrdering).\npublic struct NebulaFlag: ~Copyable, Sendable {\n    @usableFromInline internal let _atomic: Atomic<Bool>\n    public init(_ value: Bool) { _atomic = Atomic(value) }\n    public func load() -> Bool { _atomic.load(ordering: .acquiring) }\n    public func set(_ value: Bool) { _atomic.store(value, ordering: .releasing) }\n    public func compareExchange(expected: Bool, desired: Bool) -> Bool {\n        // Atomic.compareExchange has a single-ordering overload (AtomicUpdateOrdering):\n        _atomic.compareExchange(expected: expected, desired: desired,\n                                 ordering: .acquiringAndReleasing).exchanged\n    }\n}\n\n/// Instance-scoped idempotent one-time work. For GLOBAL one-time setup prefer the\n/// once-token `static let _x: Void = { ... }()` pattern (swift_once) — no primitive.\npublic struct NebulaOnce: ~Copyable, Sendable {\n    @usableFromInline internal let _done: Mutex<Bool>\n    public init() { _done = Mutex(false) }\n    public func runOnce(_ body: @Sendable () -> Void) {\n        let shouldRun = _done.withLock { d -> Bool in\n            guard !d else { return false }; d = true; return true\n        }\n        if shouldRun { body() }   // body runs outside the lock\n    }\n}\n```\n\n### API surface\n\n| Symbol | Kind | Purpose |\n|---|---|---|\n| `NebulaLocked<Value>` | public struct (~Copyable, Sendable) | Mutex wrapper for shared mutable state; `withLock`/`withLockIfAvailable` (sending); held as `let` |\n| `NebulaFlag` | public struct (~Copyable, Sendable) | Atomic<Bool> wrapper for a single thread-safe flag (enabled/disabled/registered) |\n| `NebulaOnce` | public struct (~Copyable, Sendable) | Instance-scoped idempotent one-time work via Mutex<Bool>; global once uses static-let instead |\n| `NebulaConcurrency` | DocC article | Central rules (Sendable-by-derivation, @Sendable handlers, no NSLock/DispatchQueue/nonisolated(unsafe), once-token, region isolation before @unchecked, typed throws, correct ordering-type usage, no SwiftUI constructs) |\n\n### Availability gating\n\nNone required at the Nebula `.v26` floor — every primitive ships at or below iOS 18 / macOS 15 / watchOS 11 / tvOS 18 / visionOS 2 (SwiftStdlib 6.0). The only above-floor item is [SE-0470](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0470-isolated-conformances.md) (Swift 6.2, experimental feature flags) — flagged for Nebula 27.\n\n### How this differs from Cosmos (SwiftUI sibling)\n\nCosmos threads cross-cutting concerns through `@Entry` environment values + `@Observable @MainActor` runtime theme. Nebula has NO SwiftUI, so: no `@Entry` (configurations are constructed/passed explicitly), no `@Observable`/`@MainActor` runtime theme (a foundation doesn't own UI-thread affinity), no `@Environment`. The Sendable-struct + `@Sendable`-handler + `.with*` builder contract is reused verbatim (see [[nebula-logging]], [[nebula-errors]]). Note Cosmos' `CosmosErrorEvent` deliberately avoids storing an `any Error` because `any Error` is not `Sendable` — Nebula instead uses typed throws ([SE-0413](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0413-typed-throws.md)) so the concrete `NebulaError` is `Sendable`.\n\n## Apple patterns adopted\n\n- **Sendable marker protocol with derived conformance** for all public value types ([SE-0302](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0302-concurrent-value-and-concurrent-closures.md)); `@Sendable` handler closures in every configuration, defaulting to a capture-free no-op.\n- **`@unchecked Sendable`** reserved for stdlib primitives (`Mutex`/`Atomic` are themselves `@unchecked Sendable` in the stdlib) and documented Apple-framework bridges — never authored on Nebula types.\n- **`Mutex<T>`/`Atomic<T>` from `import Synchronization`** as the only shared-mutable-state primitives ([SE-0433](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0433-mutex.md)), replacing `NSLock`/`DispatchQueue`; always `let` (`Mutex`/`Atomic` are `~Copyable` + `@_staticExclusiveOnly`). Use the correct ordering struct per operation: `AtomicLoadOrdering` for `load`, `AtomicStoreOrdering` for `store`, `AtomicUpdateOrdering` for `compareExchange`/`exchange`.\n- **Once-token pattern**: `static let _x: Void = { ... }()` side-effect initializer runs exactly once via [`swift_once`](https://github.com/swiftlang/swift/blob/main/stdlib/public/runtime/Once.cpp) (Darwin: `dispatch_once_f`) — no lock primitive.\n- **Region-based isolation ([SE-0414](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0414-region-based-isolation.md))** as the first recourse before `@unchecked Sendable` when transferring non-Sendable values.\n- **Typed throws ([SE-0413](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0413-typed-throws.md))**: `NebulaError` is a concrete `Sendable: Error`; `throws(NebulaError)` for single-known-failure APIs, untyped `throws` otherwise, `throws(Never)` for infallible.\n- **Actors ([SE-0306](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0306-actors.md))** for genuinely shared mutable state; NO `@MainActor`/`@globalActor` default isolation (non-SwiftUI).\n- **`Clock`/`Duration`/`ContinuousClock`/`SuspendingClock`** (Sendable, `_Concurrency`) for the [[nebula-standardize-measure|Measure]] subsystem instead of `Date`+`DispatchTime`.\n- **`@isolated(any)` ([SE-0431](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0431-isolated-any-functions.md))** awareness for handler-closure ergonomics where dynamic isolation matters.\n- **`sending` results/parameters ([SE-0430](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0430-transferring-parameters-and-results.md))** if Nebula exposes async APIs transferring non-Sendable values. (The keyword is `sending`, not `transferring` — verified against the shipped stdlib.)\n- **Cosmos-style Sendable-struct + `@Sendable`-handler + fluent `.with*` builder contract**, WITHOUT `@Entry`/`@Observable`/`@Environment` (foundation, not SwiftUI). [WWDC21 \"Protect mutable state with Swift actors\"](https://developer.apple.com/videos/play/wwdc2021/10133/) is the canonical Apple guidance.\n\n## Risks & open questions\n\n- **`~Copyable` propagation.** `NebulaLocked`/`NebulaFlag`/`NebulaOnce` are `~Copyable` (because `Mutex`/`Atomic` are) → they cannot be stored by value in an ordinary `Copyable` struct; they must be `let` globals or held inside actors. Verify no Nebula public API accidentally requires storing a `NebulaLocked` in a `Copyable` struct. The ~Copyable + Sendable wrapper conformance matches the verified stdlib signatures but must be compile-validated once the module exists.\n- **Ordering-type correctness (corrected).** `Atomic.load` takes `AtomicLoadOrdering` (only `.relaxed`/`.acquiring`/`.sequentiallyConsistent`); do NOT pass `.acquiringAndReleasing` to `load` (it is an `AtomicUpdateOrdering` case). `store` takes `AtomicStoreOrdering`; `compareExchange`/`exchange` take `AtomicUpdateOrdering`. There is **no single `Ordering` enum** — three frozen Sendable structs. The original draft's `NebulaFlag.load(ordering: .acquiringAndReleasing)` was invalid and is corrected to `.acquiring` (pairs with the `.releasing` store).\n- **`sending`, not `transferring` (resolved).** `Mutex.withLock` is `(_ body: (inout sending Value) throws(E) -> sending R) throws(E) -> sending R`; `init` is `(_ consuming sending Value)`. Verified in the stdlib source and binary manglings. Use `sending` in all wrapper forwarding signatures.\n- **`NebulaOnce.runOnce` runs the body OUTSIDE the lock.** If two callers race, both pass the guard but only one flips `done`; the loser's body still runs once but after the winner. Acceptable for idempotent setup; the body itself must be idempotent or internally guarded if not safe to run twice concurrently.\n- **`Atomic` requires `Value: AtomicRepresentable`** (only primitives). `NebulaFlag` uses `Bool` (fine); a future `NebulaAtomic<T>` for arbitrary `T` would need `Mutex`, not `Atomic`.\n- **[SE-0470](https://github.com/swiftlang/swift-evolution/blob/main/proposals/0470-isolated-conformances.md) (Swift 6.2, experimental feature flags)** is above the Nebula 26 floor — any code that accidentally relies on isolated-conformance inference fails without the experimental flag. Keep all conformances `nonisolated`.\n- **No `@MainActor`.** APIs invoked from main-thread callbacks must not assume main-actor isolation; handlers must be `@Sendable` and self-contained.\n- **Open:** ship `NebulaLocked`/`NebulaFlag` wrappers, or use `Mutex`/`Atomic` directly from `import Synchronization` and keep only `NebulaOnce` + a rules DocC article? Wrappers cost `~Copyable` propagation; direct use costs Nebula-prefix inconsistency. Decision needed before scaffolding.\n- **Open:** should `NebulaOnce.runOnce` accept a throwing or `async` body? Current design is synchronous non-throwing.\n- **Open:** typed-throws-by-default policy (most public APIs `throws(NebulaError)`) vs untyped throws + typed only in the errors subsystem. Needs an ADR for Nebula 26.\n- **Open:** where do shared `let NebulaLocked<NebulaStandards>` / the shared default logger live — top-level globals (once-token-initialized) or inside an `actor`? Globals are fine if `let` and `Sendable`; confirm no Nebula API needs to COPY the container.\n\n## Sources\n\n- [Synchronization — Apple Developer
